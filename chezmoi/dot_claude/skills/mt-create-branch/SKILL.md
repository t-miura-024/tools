# ブランチ作成

現在の git 差分を確認し、変更内容に基づいて適切なプレフィックスとブランチ名を生成して、新しいブランチを作成する。

## 🧠 前提知識

- ベースブランチに応じてプレフィックスを自動決定する（`main`/`master` → `hotfix/`、それ以外 → `feature/`）
- ブランチ名は英語のケバブケースで、変更内容を端的に表す名前にする
- 関連 Skill: mt-check-git-diff（変更内容の確認に使用）

## 🏃 ステップ

1. `git branch --show-current` を実行して、現在のブランチを確認する。
    - このブランチが新しいブランチのベースブランチとなる。

2. **mt-check-git-diff Skill** を実行して、現在の変更内容を確認する。
    - 変更がない場合は処理を終了する。

3. 現在のブランチに応じてプレフィックスを決定する：
    - `main` または `master` ブランチの場合 → `hotfix/`
    - それ以外のブランチの場合 → `feature/`

4. 変更内容を分析して、適切なブランチ名を生成する。
   ブランチ名は以下のルールに従う：
    - 英語のケバブケース（小文字、単語間はハイフン）で記述
    - 変更内容を端的に表す名前にする
    - 例: `fix-login-validation`, `add-user-profile`, `update-api-response`

5. ユーザーに以下の情報を提示して、ブランチ作成の確認を求める：
    - ベースブランチ（現在のブランチ）
    - 作成するブランチ名（プレフィックス付き）

    ユーザーがブランチ名の修正を希望する場合は、修正後の名前を使用する。
    ユーザーが中止を選択した場合は処理を終了する。

6. `git checkout -b [プレフィックス]/[ブランチ名]` を実行して、新しいブランチを作成する。
    - 同名のブランチが既に存在する場合はエラーを報告して処理を終了する。

7. ブランチ作成が完了したら、実行結果をユーザーに報告する。
    - 作成されたブランチ名
    - ベースブランチ

## ✅ 完了条件

- 新しいブランチが作成され、チェックアウトされている

## 📦 アウトプット

- 作成されたブランチ名（プレフィックス付き）
- ベースブランチ名
