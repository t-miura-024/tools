# Git Diff確認

現在の git 変更を包括的に分析するための Skill。

## 🧠 前提知識

- `git status --short` で変更状況を確認
- `git diff` で追跡済みファイルの詳細差分を取得
- 未追跡ファイル（`??`）は `git diff` では取得できないため、Read ツールで直接読み取る

## 🏃 ステップ

`TodoWrite` ツールを使って以下の TODO リストを作成し、進捗を管理する：

1. 変更状況の確認
2. 変更ファイル一覧の取得
3. 各ファイルの詳細 diff 取得

### ステップ1: 変更状況の確認

```bash
git status --short
```

- **変更なし**: 「変更がありません」と報告して終了
- **変更あり**: ステップ 2 へ進む

### ステップ2: 変更ファイル一覧の取得

ステップ 1 の `git status --short` の出力を変更ファイル一覧として使用する。

変更タイプ（先頭の記号）：

- `M` - 変更（追跡済みファイルの修正）
- `A` - 追加（ステージ済みの新規ファイル）
- `D` - 削除
- `R` - リネーム
- `??` - 未追跡（新規ファイル、まだ git add されていない）

### ステップ3: 各ファイルの詳細diff取得

ステップ 2 の各ファイルに対して、変更タイプに応じた方法で詳細を取得する：

- **追跡済みファイル**（M, D, R）: `git diff [ファイルパス]` で取得する
- **未追跡ファイル**（??）: `git diff` では取得できないため、新規ファイルとして報告する。内容の確認が必要な場合は Read ツールを使用する

## ✅ 完了条件

- すべての変更ファイルの詳細 diff（または新規ファイルの内容）が取得されている

## 📦 アウトプット

呼び出し元に以下の情報を返す：

- 変更状況（`git status --short` の出力）
- 変更ファイル一覧（変更タイプ付き）
- 各ファイルの詳細 diff
