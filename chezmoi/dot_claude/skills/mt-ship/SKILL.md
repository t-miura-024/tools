# 一括提出

現在の変更内容をもとに、ブランチ作成 → コミット・プッシュ → PR 作成を一気通貫で実行する。
各フェーズは既存の Skill に委譲する。

## 🧠 前提知識

- `TodoWrite` ツールを使って TODO リストを作成し、進捗を管理しながら作業する
- コミット・プッシュ実行時の権限ルール（`required_permissions: ["all"]`）は委譲先 `mt-commit-push` Skill の注意事項に従う

## 🏃 ステップ

### 1. ブランチ作成

Read ツールで **mt-create-branch Skill**（`~/.cursor/skills/mt-create-branch/SKILL.md`）を読み込んでから実行する。

- ブランチが作成されなかった場合（変更なし、ユーザー中止など）は処理を終了する。

### 2. コミット・プッシュ

Read ツールで **mt-commit-push Skill**（`~/.cursor/skills/mt-commit-push/SKILL.md`）を読み込んでから、**all スコープ**で実行する。

- コミット・プッシュが完了しなかった場合はエラーを報告して処理を終了する。

### 3. PR 作成

Read ツールで **mt-submit-pr Skill**（`~/.cursor/skills/mt-submit-pr/SKILL.md`）を読み込んでから実行する。

- PR が作成されなかった場合はエラーを報告して処理を終了する。

### 4. 完了報告

すべてのフェーズが完了したら、ユーザーに最終報告する：

- 作成されたブランチ名
- コミットハッシュとメッセージ
- PR の URL

## ✅ 完了条件

- ブランチが作成されている
- 変更がコミット・プッシュされている
- ドラフト PR が作成されている
- ユーザーに最終報告が提示されている
