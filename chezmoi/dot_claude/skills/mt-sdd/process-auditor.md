# Process Auditor

## 監査サマリ

- **入力**: 対象フェーズまでの全中間生成物
- **出力**: 監査サマリ（レビューレポートの末尾に追記）
    - **プロセス健全性**: Healthy / Warning / Critical
    - **検出事項**: 具体的な問題点
    - **推奨アクション**: 人間が確認すべきポイント

## 監査観点

### 成果物の完全性

| チェック項目 | 判定基準 |
|-------------|----------|
| セクション充足 | テンプレートの全セクションが記述されているか |
| 空欄・TODO 残存 | 未記入のセクションや TODO コメントが残っていないか |
| 具体性 | 抽象的な記述に留まらず、具体的な内容が記述されているか |

### 入力の反映度

| チェック項目 | 判定基準 |
|-------------|----------|
| 要件トレーサビリティ | `requirements.md` の各要件が後続成果物で追跡可能か |
| 情報の欠落 | 前フェーズの成果物にあった情報が次の成果物で失われていないか |
| 文脈の保持 | 要件の背景・意図が仕様や計画に正しく反映されているか |

### レビューの反映度

| チェック項目 | 判定基準 |
|-------------|----------|
| Critical 対応 | 自動修正ループで Critical 指摘が解消されたか |
| 修正の品質 | Critical 指摘への対応が表面的でなく、本質的な修正か |
| 副作用の有無 | 修正によって新たな問題が生じていないか |

### プロセス逸脱

| チェック項目 | 判定基準 |
|-------------|----------|
| フェーズ順守 | SDD フローの手順がスキップされていないか |
| 成果物間の整合 | 成果物間で矛盾する記述がないか |
| レビュー実施 | 全レビュアーの結果が集約されているか |

### UCR 処理の適切性

| チェック項目 | 判定基準 |
|-------------|----------|
| UCR の検出漏れ | レビューコメントや SubAgent の出力に上流変更の必要性を示唆する内容があるのに、UCR として処理されていないケースがないか |
| 連鎖更新の実施 | spec.md を更新した場合、implementation-plan.md への影響が評価されているか |
| 変更ログの記録 | 処理された UCR が `appendix-change-log.md` に記録されているか |
| 成果物の整合性維持 | UCR 処理後、関連する全成果物間の整合性が保たれているか |
