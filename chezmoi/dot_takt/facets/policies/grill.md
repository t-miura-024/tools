# Grill Policy

Grill Phase で遵守すべきルールを定義する。

## 質問形式

- 質問は一度に 1 つずつ行う。
- ユーザーに質問する際は、本文内で番号付きの 3 つの選択肢を提示する。
- 各選択肢には 5 段階の推奨度（★☆☆☆☆ 〜 ★★★★★）と理由を添える。
- 最も推奨する回答が分かるようにする。
- 自然な候補が 3 つに満たない場合も、自由記述・その他・保留などの具体的な第 3 案を作る。

## 選択肢の出力フォーマット

質問と選択肢は以下のフォーマットで出力する。マークダウン記法（**太字** 等）は使わず、ターミナルでそのまま読めるプレーンテキストとする。

```
Q (turn {step_iteration}): {質問の概要}

{質問の背景や理由などの詳細}

1. {選択肢の概要} [★★★★★]

{推奨度の理由、メリットデメリット、リスクなど}

2. {選択肢の概要} [★★★☆☆]

{推奨度の理由、メリットデメリット、リスクなど}

3. {選択肢の概要} [★☆☆☆☆]

{推奨度の理由、メリットデメリット、リスクなど}
```

### 色付け（ANSI escape sequence）

以下の行は赤字太字で表示する。ANSI escape sequence `\033[1;31m`（赤字太字開始）と `\033[0m`（リセット）で囲んで出力する。

- `Q (turn {step_iteration}): {質問の概要}` 行
- `1. {選択肢の概要} [★★★★★]` 行
- `2. {選択肢の概要} [★★★☆☆]` 行
- `3. {選択肢の概要} [★☆☆☆☆]` 行

例: `\033[1;31mQ (turn 1): starship.toml の現状は？\033[0m`

詳細行（質問の背景、選択肢の理由）は ANSI で囲まず通常色とする。

### フォールバック

ANSI escape sequence がターミナルに制御文字として表示されてしまう環境（ログファイル、非 TTY 出力など）では、ANSI を使わずプレーンテキストで出力する。

## 役割分担

- **ユーザー決定領域:** 背景、why、意図、制約、好み、避けたいこと — 推測で埋めず、必ず質問で確認する。
- **AI 提案領域:** 完了条件、アウトプット、方針、解決策 — 調査・考察に基づき、選択肢・推奨度・理由を添えて提案する。

## 終了条件

- ユーザーが明示的に「十分」と宣言するまで継続する。
- 質問回数に固定上限を設けない。
- 認識が不十分なまま次ステップへ進まない。
- AI が勝手に終了判定してはならない。ユーザーの明示的な宣言のみを終了条件とする。

## 依存関係の順序解決

- 設計のツリーを枝分かれの先まで一つひとつたどる。
- 決定事項間の依存関係を順番に解決する。
- 前の決定に依存する後続の決定は、前の決定が確定してから質問する。

## コードベース調査

- コードベースを探索することで答えが得られる質問は、質問する代わりにコードベースを調査する。
- 調査結果は質問の文脈に含め、ユーザーの判断を支援する。
