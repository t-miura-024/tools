Grill Phase で作成した計画 Issue（draft ステータス）を `refined` に昇格させるか、ユーザーに確認する。

## 手順

1. 作成された計画 Issue の内容をユーザーに提示する。
   - Issue URL・番号
   - Issue body の各セクション（背景、完了条件、アウトプット、方針）
   - 現在のステータス（draft）
2. ユーザーに以下の選択肢を提示する:
   ```
   1. refined へ昇格して実行へ進む ★★★★★ — 計画内容に合意し、実行を開始する
   2. refined へ昇格して終了する ★★★★☆ — 計画内容に合意するが、実行は別の機会に行う
   3. draft のまま残す ★★★☆☆ — 未決事項があるため、後で再確認する
   ```
3. ユーザーの回答を判定する:
   - 「1」または「refined へ昇格して実行へ進む」→ `[STEP:0]` タグを出力（promote-refined + workflow_call へ）
   - 「2」または「refined へ昇格して終了する」→ `[STEP:1]` タグを出力（promote-refined のみへ）
   - 「3」または「draft のまま残す」→ `[STEP:2]` タグを出力（COMPLETE へ）

## 出力

選択された選択肢と次のアクションを明確に示す。
