実装が計画のスコープ範囲内に収まっているかを評価する。

## 手順

1. 計画 Issue の `## 📦 アウトプット` と `## 🧭 方針` を読み、スコープ境界を把握する。
2. `{previous_response}` の実行レポートと、実行ステップで作成・編集したファイル群を確認する。
3. 以下を評価する:
   - スコープ外の変更（計画にないファイル編集、不要な機能追加、関連しない修正）はないか。
   - 計画の `## 🧭 方針` でスコープ外化されたものが遵守されているか。
   - スコープ内だが計画に明記されていない副作用（設定ファイルの変更、依存関係の更新など）がないか。
4. スコープ外の変更が発見された場合は must 指摘とする。
5. 計画に明記されていない副作用がある場合は should 指摘とする。
6. 過去のレビュー指摘（`## 🔍 レビュー`）を確認し、既出の指摘を踏まえてレビューする。
7. output-contracts/review-verdict.md に従って結果を出力する。

## 終了判定

- must 指摘が 0 件の場合: `approved` タグを出力。
- must 指摘が 1 件以上の場合: `needs_fix` タグを出力。
