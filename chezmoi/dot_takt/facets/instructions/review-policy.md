実装が計画の方針と整合しているかを評価する。

## 手順

1. 計画 Issue の `## 🧭 方針` を読み、進め方の原則、AI 判断範囲、Human Gate の境界を把握する。
2. `{previous_response}` の実行レポートと、実行ステップで作成・編集したファイル群を確認する。
3. 以下を評価する:
   - `## 🧭 方針` の「進め方の原則」に違反する実装アプローチはないか。
   - `## 🧭 方針` の「AI 判断範囲・Human Gate」でユーザー確認が必要な境界を越えた実行はないか。
   - 方針の変更がユーザー合意なく行われていないか。
   - 方針の精神には沿っているが文字通りには従っていない部分はないか。
4. 方針違反や Human Gate 越えを must 指摘とする。
5. 方針の精神には沿っているが文字通りに従っていない場合、should 指摘として改善を提案する。
6. 過去のレビュー指摘（`## 🔍 レビュー`）を確認し、既出の指摘を踏まえてレビューする。
7. output-contracts/review-verdict.md に従って結果を出力する。

## 終了判定

- must 指摘が 0 件の場合: `approved` タグを出力。
- must 指摘が 1 件以上の場合: `needs_fix` タグを出力。
