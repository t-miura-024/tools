実装が目的に対して本質的で効率的な解決となっているかを評価する。

## 手順

1. 計画 Issue の `## 💭 背景` と `## 🧭 方針` を読み、本来の目的を把握する。
2. `{previous_response}` の実行レポートと、実行ステップで作成・編集したファイル群を確認する。
3. 以下を評価する:
   - 実装は本来の目的に沿っているか。
   - 完了条件を満たすための最小本質的解決か。
   - 過剰な抽象化、不必要な機能、目的と無関係な変更はないか。
   - よりシンプルで本質的な代替案があるか。
4. 過去のレビュー指摘（`## 🔍 レビュー`）を確認し、既出の指摘を踏まえてレビューする。
5. output-contracts/review-verdict.md に従って結果を出力する。

## 終了判定

- must 指摘が 0 件の場合: `approved` タグを出力。
- must 指摘が 1 件以上の場合: `needs_fix` タグを出力。
