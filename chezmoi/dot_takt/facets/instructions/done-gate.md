レビューで all approved となった後、ユーザーに done 化を確認する。

## 手順

1. レビュー結果をサマリとして提示する:
   - 各観点のレビュー結果（approved / needs_fix）。
   - should / want 指摘の有無と対応状況。
2. 完了条件の充足状況を確認する:
   - `## ✅ 完了条件` のすべての条件が満たされていることを確認。
   - 未解決の `## 🐿️ メモ` の `🤔 論点` がないことを確認。
3. ユーザーに以下の選択肢を提示する:
   ```
   1. Done にする ★★★★★ — 完了条件を満たし、計画を完了とする
   2. 追加の指示や質問がある ★★★☆☆ — 追加の対応や確認が必要
   ```
4. ユーザーの回答を判定する:
   - 「1」または「Done にする」→ `[STEP:0]` タグを出力（promote-done へ）。
   - 「2」または「追加の指示や質問がある」→ `[STEP:1]` タグを出力（execute へ戻る）。

## 出力

選択された選択肢と、Done の場合は Issue URL・番号・最終ステータスを報告する。
