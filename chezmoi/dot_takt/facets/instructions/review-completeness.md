計画 Issue の完了条件が完全に満たされているかを評価する。

## 手順

1. 計画 Issue の `## ✅ 完了条件` を読み、すべての条件を列挙する。
2. `{previous_response}` の実行レポートと、実行ステップで作成・編集したファイル群を確認する。
3. 各完了条件について、Yes/No で判定可能な形で充足を確認する:
   - 成果物の存在（`## 📦 アウトプット`）は存在確認。
   - 完了条件（`## ✅ 完了条件`）は充足確認。
   - 両者を区別して評価する。
4. 未充足の完了条件がある場合は must 指摘とする。
5. 完了条件が「よくする」「整理する」などの曖昧な表現の場合は should 指摘として測定可能な形への修正を提案する。
6. 過去のレビュー指摘（`## 🔍 レビュー`）を確認し、既出の指摘を踏まえてレビューする。
7. output-contracts/review-verdict.md に従って結果を出力する。

## 終了判定

- must 指摘が 0 件の場合: `approved` タグを出力。
- must 指摘が 1 件以上の場合: `needs_fix` タグを出力。
