レビューで指摘された must / should 指摘に基づいて、実装を修正する。

## 手順

1. `{previous_response}` に含まれるレビュー結果（各観点の review-verdict）を読み込む。
2. must 指摘をすべて抽出する。
3. 各 must 指摘に対して、指摘内容と修正案を確認し、修正を実装する。
4. should 指摘のうち、対応可能なものも併せて修正する。
5. 修正内容を確認し、必要なら検証コマンドや linter を実行する。
6. output-contracts/execution-report.md に従って修正レポートを出力する。
7. GitHub Issue body の `## 🐢 履歴` に修正サマリを追記する。

## 終了判定

- must 指摘への修正が完了した場合: `[STEP:0]` タグを出力（review へ戻る）。
- 修正が不可能な場合（スコープ外、計画変更が必要など）: `[STEP:1]` タグを出力（ABORT または計画修正へ）。

## スコープ遵守

- 修正はレビュー指摘の範囲内に留める。
- 指摘への対応でスコープを超える変更が必要な場合は、実行を止めてユーザーに確認する。
