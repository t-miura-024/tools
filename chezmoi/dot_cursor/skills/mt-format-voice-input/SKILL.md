# 音声入力テキストの整形

音声入力されたテキストを読みやすく整形するスキルです。

## 🧠 前提知識

### 連鎖呼び出し

ユーザーが**明示的に**本スキルと別スキルを続けて実行するよう指示した場合（例: 「mt-format-voice-input のあと nt-create-task」「整形してから nt-record-idea」）は、次の契約に従う。

- まず本スキルで整形を完了し、**整形後のテキスト**を、ユーザーが指定した次スキルへの入力として渡す。
- **後段のスキルは `mt-format-voice-input` を再実行しない**。入力は既に整形済みとみなす。
- ユーザーが連鎖を指示していない場合、他スキルは本スキルを内包しない（各スキルは渡されたテキストをそのまま扱う）。

## 🏃 ステップ

入力されたテキストに対して、以下の整形を実施してください：

### 1. 改行の挿入

- 文の区切り（句点「。」の後）で適切に改行を入れる
- 話題が変わる箇所で段落を分ける
- 長い文が続く場合は、意味のまとまりで改行を入れる

### 2. 誤変換の修正

- **[conversion-history.md](conversion-history.md) を参照し、過去に修正したパターンと同じ誤変換があれば優先的に修正する**
- 文脈から音声認識の誤変換と思われる箇所を推測して修正する
- 同音異義語の誤りを文脈に合わせて修正する
- 固有名詞の誤変換は前後の文脈から推測する

### 3. 句読点の補完

- 欠落している句点「。」や読点「、」を補う
- 疑問文には「？」を付ける

## ✅ 完了条件

- 入力テキストに対して、改行挿入・誤変換修正・句読点補完の 3 観点すべてが適用されている
- 整形後テキストが出力されている
- 修正箇所がある場合は「修正箇所」リストが提示されている（修正なしなら省略可）
- `conversion-history.md` への追加件数（または「なし」）が必ず報告されている
- 連鎖呼び出し指定がある場合、整形後テキストが次スキルへの入力として引き渡されている

## 📦 アウトプット

以下の順序で出力してください：

1. **整形後のテキスト**をそのまま出力
2. **修正箇所**がある場合は、以下の形式で記載（修正なしなら省略）：

    ```markdown
    ---

    **修正箇所:**
    - 「〇〇」→「△△」（理由: 文脈から□□と推測）
    ```

3. **誤変換の修正履歴の更新**（必須）
    - 新しい誤変換パターンが見つかった場合は、[conversion-history.md](conversion-history.md) のテーブルに追加する
    - 追加した場合は「履歴に追加: ○件」と明記する
    - 追加がない場合は「履歴への追加: なし」と明記する

## ⚠️ 注意事項

- 元の意味や意図を損なわないよう、最小限の修正にとどめる
- 話し言葉の雰囲気は残しつつ、読みやすさを優先する
- 明らかな誤変換のみ修正し、曖昧な場合は元のままにする
- 修正に自信がない場合は、修正箇所に「（推測）」と明記する
