# セッション引き継ぎ

現在の会話を、別エージェントが継続できる引き継ぎ文書にまとめる。

## 🏃 ステップ

### 1. 次セッションの焦点を確認する

- ユーザー引数があれば、それを次セッションの焦点として扱う
- なければ、現在の未完了作業の継続を焦点にする

### 2. 既存成果物を参照で拾う

- 仕様、計画、ADR、Issue、コミット、差分に既にある内容は複製しない
- パスまたは URL で参照する

### 3. 引き継ぎ文書を書く

OS の一時ディレクトリへ保存する。ワークスペース内には置かない。

含める内容:

1. 目的と現状
2. 完了済み
3. 未完了と再開位置
4. 重要な決定と制約
5. 参照すべき成果物
6. 推奨 Skill
7. 次の具体アクション

### 4. 秘匿情報を除去する

- API キー、パスワード、個人情報を含めない
- 必要なら「要再設定」とだけ書く

### 5. パスを報告する

- 絶対パスをユーザーに伝える

## ✅ 完了条件

- 一時ディレクトリに引き継ぎ文書がある
- 既存成果物の重複がなく、参照で辿れる
- 秘匿情報が含まれていない

## 📦 アウトプット

- OS 一時ディレクトリ上の Markdown 引き継ぎ文書
