# ブランチ差分確認

現在のブランチとベースブランチの差分を包括的に取得するための Skill。
ベースブランチの自動検出から各ファイルの詳細 diff 取得までを一貫して行う。

## 🧠 前提知識

- `git log --oneline --graph --decorate --all` でブランチ分岐構造を可視化
- `git merge-base` でベースブランチとの正確な分岐点を取得
- `origin/[ベースブランチ]..HEAD` を直接使うと、ローカルブランチが origin より遅れている場合に不正確な差分が出るため、マージベースを経由する

## 🏃 ステップ

`TodoWrite` ツールを使って以下の TODO リストを作成し、進捗を管理する：

1. ベースブランチの検出
2. マージベースの取得
3. 差分ファイル一覧の取得
4. 変更統計の取得
5. 各ファイルの詳細 diff 取得

### ステップ1: ベースブランチの検出

`git log --oneline --graph --decorate --all -n 50` を実行して、ブランチの分岐構造を可視化し、現在のブランチの分岐元（ベースブランチ）を特定する。

50 件で分岐元が特定できない場合は、段階的に取得数を増やして再試行する：

- `-n 50` で特定できない → `-n 100` で再試行
- `-n 100` で特定できない → `-n 200` で再試行
- `-n 200` で特定できない → `-n 300` で再試行
- それでも特定できない場合 → ユーザーにベースブランチを直接確認する

特定したブランチを以降の手順でベースブランチとして使用する。

### ステップ2: マージベースの取得

```bash
git merge-base origin/[ベースブランチ] HEAD
```

ベースブランチとの正確な分岐点（マージベースコミット）を取得する。
取得したコミットハッシュを以降のステップで `[マージベース]` として使用する。

### ステップ3: 差分ファイル一覧の取得

```bash
git diff --name-status [マージベース]..HEAD
```

変更タイプ：

- `A` - 追加
- `M` - 変更
- `D` - 削除
- `R` - リネーム

- **差分なし**: 「ベースブランチとの差分がありません」と報告して終了
- **差分あり**: ステップ 4 へ進む

### ステップ4: 変更統計の取得

```bash
git diff --stat [マージベース]..HEAD
```

追加・削除行数の合計と変更ファイル数を取得する。

### ステップ5: 各ファイルの詳細diff取得

ステップ 3 で取得した各ファイルに対して実行：

```bash
git diff [マージベース]..HEAD -- [ファイルパス]
```

## ✅ 完了条件

- ベースブランチが特定されている
- すべての変更ファイルの詳細 diff が取得されている

## 📦 アウトプット

呼び出し元に以下の情報を返す：

- ベースブランチ名
- 差分ファイル一覧（name-status）
- 変更統計（stat）
- 各ファイルの詳細 diff
