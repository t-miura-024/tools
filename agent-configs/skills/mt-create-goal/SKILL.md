# mt-create-goal

`/mt-goal` 用のゴール条件ドキュメントを作成・リファインメントするパートナーとして振る舞い、実行（`/mt-goal` 自体の起動）は行わない。実行は `/mt-goal` コマンドと `mt-goal` Skill の責務。

## 共有資材

隣接する `mt-goal/` 配下の以下を参照する。いずれかが存在しない場合は中断する。

- `goal-format.md` — ゴール条件ドキュメントの本文フォーマット
- `list-goals.ts` — 既存ゴール条件ドキュメントの一覧取得
- `transition-goal.ts` — ステータス遷移（`draft` → `refined` → `done`）

## 🏃 ステップ

### 1. Grill Phase

このゴール条件のあらゆる側面について、私たちが共通の認識に達するまで、徹底的に私に質問を投げかけてください。

条件のツリーを枝分かれの先まで一つひとつたどり、決定事項間の依存関係を順番に解決していきましょう。

ユーザーに質問する際は、本文内で番号付きの 3 つの選択肢を提示してください。各選択肢には 5 段階の推奨度（例: ★★★★☆）と理由を添え、あなたが最も推奨する回答が分かるようにしてください。自然な候補が 3 つに満たない場合も、自由記述・その他・保留などの具体的な第 3 案を作ってください。

質問は一度に 1 つずつお願いします。もしコードベースを探索することで答えが得られる質問であれば、質問する代わりにコードベースを調査してください。

#### 役割分担

- **ユーザー決定領域:** 背景、why、意図、制約、好み、避けたいこと — 推測で埋めず、必ず質問で確認する
- **AI 提案領域:** `/mt-goal 条件`、測定方法、方針、解決策 — 調査・考察に基づき、選択肢・推奨度・理由を添えて提案する。`/mt-goal` 実行時は AI が自律的に判断して進めるため、Human Gate の設定は不要

#### 終了条件

Grill Phase はユーザーが明示的に「十分」と宣言するまで継続する。質問回数に固定上限を設けない。認識が不十分なまま次ステップへ進まない。

### 2. ゴール条件ドキュメントの作成

1. `goal-format.md` に従って `tmp/mt-goal/docs/draft/yyyymmdd-[goal-name].md` にゴール条件ドキュメントを作成する
2. `## 🎯 /mt-goal 条件` セクションには、`/mt-goal <condition>` に渡せる一行の条件文字列を必ず含める
3. 作成内容をユーザーに報告し、修正があれば反映する

### 3. Refined 昇格

1. Refined 昇格の判定基準を満たす場合、ユーザーに昇格を確認する。承認されたら `transition-goal.ts` で `tmp/mt-goal/docs/refined/` へ移動する
2. 不足があれば `draft` のまま残し、次回確認事項をドキュメント内に明記する

### 4. `/mt-goal` 実行への誘導

`refined` 昇格後、次のステップとして `/mt-goal` コマンドを実行することをユーザーに案内する。

- `## 🎯 /mt-goal 条件` セクションの一行条件をコピーし、`/mt-goal <condition>` として実行できることを伝える
- ドキュメントパスをそのまま渡すこともできることを伝える（例: `/mt-goal tmp/mt-goal/docs/refined/yyyymmdd-[goal-name].md`）
- 必要に応じて、状態ファイル `tmp/mt-goal/state.json` の初期化方法や設定方法を `mt-goal` Skill の内容に基づいて案内する
- 実行は `mt-create-goal` の責務外なので、直接 `/mt-goal` を起動しない

## ✅ 完了条件

- ゴール条件ドキュメントが `goal-format.md` に従って作成されている
- `## 🎯 /mt-goal 条件` セクションに `/mt-goal <condition>` へ渡せる一行の条件文字列が含まれている
- ユーザーがドキュメント内容を承認し、`refined` への昇格が完了している、または `draft` のまま次回確認事項が明記されている
- `/mt-goal` 実行への誘導が行われている

## 📦 アウトプット

- 作成・更新されたゴール条件ドキュメント（`tmp/mt-goal/docs/draft/` または `tmp/mt-goal/docs/refined/`）
- 次のアクション（`/mt-goal` 実行）に関するユーザーへの案内

## ⚠️ 注意事項

- 直接 `/mt-goal` を起動しない。`mt-create-goal` は条件の文書化・承認までが責務。
- `draft` のドキュメントを `/mt-goal` 実行に使わせない。先に `refined` へ昇格させる。
- `goal-format.md` の本文を `SKILL.md` に重複させない。
- 状態遷移は `transition-goal.ts` を使い、`mv` や手動移動をしない。
