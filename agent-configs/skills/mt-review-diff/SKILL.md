# 差分レビュー

ブランチ差分やコミットされていない差分を、マクロ・ミクロの 2 段階でレビューし、指摘と提案を明確に分離して出力する。

## 🧠 前提知識

- レビュー観点・コメント分類: [code-review-criteria.md](code-review-criteria.md) を参照
- 出力フォーマット: [REVIEW-TEMPLATE.md](REVIEW-TEMPLATE.md) を参照
- 関連 Skill: mt-check-branch-diff（ブランチ差分の取得に使用）
- 変更ファイル 6 件以上は SubAgent 並列実行に切り替える
- レビュー前に背景（目的・制約・意図）を抽出する。抽出元はセッション文脈を基本とし、ブランチ差分レビュー時は既存 PR があればその内容も参照する。不十分な場合はユーザーに確認または提供を依頼する。確定した背景要約はマクロレビューおよびレポートの「採用した背景」に反映する

## 🏃 ステップ

`TodoWrite` ツールを使って以下の TODO リストを作成し、進捗を管理する：

1. レビュー対象のヒアリング
2. 背景の抽出・確認
3. 差分の取得
4. 変更規模の判定
5. レビュー実施
6. レビューレポート生成

### ステップ1: レビュー対象のヒアリング

レビュー開始前に、本文でレビュー対象の選択肢を番号付きで提示し、ユーザーのテキスト回答から対象を判断する。

> レビュー対象を選択してください。

選択肢:

- a) ブランチ差分のみ（ベースブランチとの差分）
- b) コミットされていない差分のみ（ステージ済みのみ）
- c) コミットされていない差分のみ（すべて: ステージ済み + 未ステージ）
- d) ブランチ差分 + コミットされていない差分（ステージ済みのみ）
- e) ブランチ差分 + コミットされていない差分（すべて: ステージ済み + 未ステージ）

### ステップ2: 背景の抽出・確認

差分を読む前に、レビュー判断に必要な背景（目的・制約・意図）を確定する。特定の Vault パス・Skill 名・ファイルパスへの自動探索は必須としない。

#### 抽出元

| 優先 | 抽出元 | 条件 |
| ---- | ------ | ---- |
| 1 | セッション文脈 | 常に実施（会話・ユーザーの説明・既に共有された文脈） |
| 2 | 既存 PR | ステップ 1 で **a / d / e**（ブランチ差分を含む）を選択した場合 |
| 3 | ユーザー指定 | ファイルパスや wikilink を明示された場合のみ `Read` 等で追加 |

#### 手順

1. **セッションから抽出**: セッション文脈をもとに、レビューに必要な背景を短い要約（3〜7 行程度）にまとめる
2. **既存 PR の確認**（ブランチ差分レビューの場合のみ）:
    - 現在ブランチを取得する: `git branch --show-current`
    - 既存 PR の有無を `gh pr list --head "$(git branch --show-current)" --json number,title,body,url --limit 1` で確認する
    - 1 件以上ヒットした場合、`title`・`body`（概要・背景・目的などのセクション）から目的・制約・意図を抽出し、セッション要約と統合する
    - PR が存在しない、または `gh` が利用できない場合はセッション要約のみで続行する（エラーでレビューを止めない）
3. **十分性の判断**: 統合後の要約だけで「なぜこの変更か」「何を達成すべきか」をマクロレビュー（[code-review-criteria.md](code-review-criteria.md) の「背景目的との整合性」）に使えるか判断する
4. **不足時の確認**: 十分でない場合、本文で次のいずれかを提示し、ユーザーのテキスト回答で確認する
    - 抽出した背景要約（セッション・PR 由来を区別して提示）を示し、「この理解で合っているか」を確認する
    - 背景情報（目的・制約・意図）の提供をユーザーに依頼する
5. **確定**: ユーザー確認または提供を経て、レビューに採用する背景要約を確定する。PR から採用した場合は要約末尾に PR URL を 1 行で添える。以降のステップではこの要約を参照する

### ステップ3: 差分の取得

ステップ 1 のヒアリング結果に応じて差分を取得する。

#### ブランチ差分を取得する場合

**mt-check-branch-diff Skill** を使用して、ベースブランチの検出から各ファイルの詳細 diff 取得までを行う。

#### コミットされていない差分を取得する場合

ヒアリングで選択されたスコープに応じたコマンドを実行する。

##### スコープ: ステージ済みのみ

```bash
# ステータス確認
git status --short

# 差分ファイル一覧
git diff --staged --name-status

# 変更統計
git diff --staged --stat

# 各ファイルの詳細diff
git diff --staged -- [ファイルパス]
```

##### スコープ: すべて（ステージ済み + 未ステージ）

```bash
# ステータス確認
git status --short

# 差分ファイル一覧
git diff HEAD --name-status

# 変更統計
git diff HEAD --stat

# 各ファイルの詳細diff
git diff HEAD -- [ファイルパス]
```

未追跡ファイルがある場合は追加で取得する：

```bash
# 未追跡ファイルの一覧
git ls-files --others --exclude-standard
```

未追跡ファイルの内容は `Read` ツールで直接読み取る。

差分が存在しない場合は「差分がありません」と報告して終了する。

#### 両方取得する場合

ブランチ差分とコミットされていない差分の両方を取得し、レビュー時に統合して扱う。

### ステップ4: 変更規模の判定

変更ファイル数に応じて実行方法を決定：

| 条件   | 判定基準            | 実行方法              |
| ------ | ------------------- | --------------------- |
| 小規模 | 変更ファイル5件以下 | 単一フローで順次実行  |
| 大規模 | 変更ファイル6件以上 | Task ツールで並列実行 |

### ステップ5: レビュー実施

ステップ 2 で確定した背景要約を常に参照する。

#### 小規模変更の場合（順次実行）

1. マクロレビュー実施（[code-review-criteria.md](code-review-criteria.md) の「背景目的との整合性」を必ず確認）
2. ミクロレビュー実施

#### 大規模変更の場合（Task ツールで並列実行）

以下のサブエージェントを Task ツールで並列起動する。各 Agent の prompt には、ステップ 2 で確定した背景要約を必ず含める。

##### Agent 1: マクロレビュー

- アーキテクチャ観点のレビュー（[code-review-criteria.md](code-review-criteria.md) の「背景目的との整合性」を必ず確認）
- 横展開候補の検出（コードベースの類似実装を検索）

##### Agent 2: ミクロレビュー

- コード詳細のレビュー（背景要約を踏まえ、目的に沿った実装かを確認）

### ステップ6: レビューレポート生成

1. 全レビュー結果を集約
2. 推奨度順にソート（高い順）
3. [REVIEW-TEMPLATE.md](REVIEW-TEMPLATE.md) のフォーマットで出力（冒頭の「📋 採用した背景（要約）」にステップ 2 で確定した内容を記載）

## ✅ 完了条件

- レビュー開始前に背景要約が確定している（ステップ 2）
- すべての差分ファイルがマクロ・ミクロの両観点でレビューされている
- レビューレポートが REVIEW-TEMPLATE.md のフォーマットで出力され、「📋 採用した背景（要約）」が記載されている

## 📦 アウトプット

[REVIEW-TEMPLATE.md](REVIEW-TEMPLATE.md) のフォーマットに従ったレビューレポート。

各コメントには以下を必ず含める：

- **推奨度**: ★★★★★（5 段階）
- **種別**: ❗ 指摘 / 💡 提案
- **レベル**: 🌍 マクロ / 🌲 ミクロ
- **重大度**: 🚨 Critical / ⚠️ Warning / ℹ️ Info
- **観点**: 絵文字付きの観点
- **内容**: 具体的な指摘/提案内容
- **メリット**: 修正/採用した場合のメリット
- **デメリット**: 修正/採用した場合のデメリット（あれば）

❗ 指摘コメントには追加で：

- **確認手順**: 指摘した事象を確認・再現するための手順

🌲 ミクロレビューのコメントには追加で：

- **ファイル**: パス:行番号
- **Before/After**: コード例（該当する場合）
